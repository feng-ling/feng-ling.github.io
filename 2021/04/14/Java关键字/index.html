    Java跳出多重循环的三种办法

    1、利用标志变量

    public class Test1 {
    public static boolean flag = true;   //标识变量
    public static int i = 1;
    public static int j = 1;

    public static void main(String[] args) {
        for (;i<10;i++){
            for(;j<10;j++){
                if(flag){
                    break;
                }
            }
            if(flag) {
                break;
            }
        }
        System.out.println("i="+i+"     j="+j);
    }
}

    2、利用try…catch…

    public class Test2 {
    public static int i=1;
    public static int j=1;
    public static int max = 10;

    public static void main(String[] args) {
        try {
            for (; i < max; i++) {
                for (; j < max; j++) {
                    throw new Exception("跳出循环");
                }
            }
        } catch (Exception e) {

        }
        System.out.println("i="+i+"     j="+j);
    }
}

    3、利用标签变量和break

    public class Test3 {
    public static int i = 1;
    public static int j = 1;
    public static int max = 10;

    public static void main(String[] args) {
        label: for (; i < max; i++) {
            for (; j < max; j++) {
                break label;
            }
        }
        System.out.println("i=" + i + "     j=" + j);
    }
}

    static关键字

    （1）static 成员变量

    静态变量属于类，可以达到全局变量的效果，他在内存中只有一个复制，所有的实例都指向同一个内存地址。

    在Java语言中，不能在方法体中定义静态变量。

    （2）static成员方法

    static方法是类的方法，在其中不能使用this和super关键字（因为，可能静态方法被调用的时候，类的对象还没被创建），只能访问所属的类的静态成员方法和变量。

    static一个很重要的用提就是实现单例模式。单例模式的特点是该类只能有一个实例，因此我们将类的构造函数声明为private类型，同时将创建类的对象的方法和对象变量声明为static。

    （3）static代码块

    JVM在加载类的时候会优先执行static代码块，有多个的时候，依次执行。

    （4）static内部类

    同面向对象中的静态内部类。

    switch（expr）

    在使用switch（expr）时，expr只能是一个枚举常量（内部由整型或者字符类型实现，enum xx{。。。}）或者一个整数表达式，整数表达式可以是byte、short、int、char类型以及他们对应的包装类类型。

    从Java7开始，switch开始支持String类型了。下面是具体解释：

    从本质上来讲，switch对字符串的支持，其实是int类型值的匹配。原理：通过case后面的String对象调用hashCode()方法，得到一个int类型的hash值，然后用着hash值来唯一标志这个case。当匹配的时候，调用expr的hashCode()函数，得到一个hash值，然后用这个值来匹配所有的case,如果没有匹配成功，则说明不存在；如果匹配成功，则调用字符串的String.equals()方法进行匹配。由此可以看出，switch。。。case字符串表达式不能为null。
    至于为什么要调用String.equals()方法是因为，hashCode()返回值相等，他们的equals（）的返回值不一定相等。

    volatile

    volatile有什么作用

    （1）被volatile修饰的变量，系统每次用到它的时候都是直接从内存中提取而不是利用缓存（编译器自动对程序进行优化，对经常访问的变量缓存起来，但是会造成线程访问出现问题）；

    （2）在使用了volatile修饰成员变量后，所有线程在任何时候看到的变量都是相同的。

    缺点：

    （1）volatile不能保证操作的原子性，所以一般不能用volatile代替sychronized；

    （2）使用volatile会组织编译器对代码进行优化，会降低程序的执行效率。
