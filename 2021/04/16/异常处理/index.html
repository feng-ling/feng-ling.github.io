    try{}ctach(){}finally{}

    finally块中的语句什么时候执行

    （1）在Java语言的异常处理中,finally块的作用就是为了保证无论出现什么情况，finally块里的代码一定会被执行。

    （2）finally块里的代码也是在return前执行的。

    （3）此外，如果try- fmally或者catch - finally中都有return, 那么finally块中的return语句将会覆盖别处的return语句，最终返回到调用者那里的是finally中return的值。

    （4）对于基本数据类型的数据，在finally块中改变return的值对返回值是没有任何影响的，而对引用类型的数据是由影响的。

    引申：出现在Java程序中的finally块是不是一定会被执行？

    在下面两种情况下不一定执行：
    （1）当程序在进入try语句块之前就出现异常时，会直接结束，不会执行finally块中的代码；

    （2）当程序在 try 块中使用System.exit(0)(推出JVM)强制退出时也不会去执行 finally 块中的代码。

    异常处理的原理是什么？

    异常是指程序运行时（非编译时）所发生的非正常情况或错误，当程序违反了语义规则时，JVM 就会将出现的错误表示为一个异常并抛出。

    这个异常可以在catch程序块中进行捕获，然后进行处理。而异常处理的目的则是为了提高程序的安全性与鲁棒性。

    Java 语言把异常当作对象来处理，并定义了一个基类 (java.lang.Throwable) 作为所有异常的父类。在Java API中，已经定义了许多异常类，这些异常类分为 Error(错误）和Exception (异常）两大类。

    运行时异常和普通异常有什么区别？

    在 Java 中，所有的异常都有一个共同的祖先java.lang包中的 Throwable类。Throwable类有两个重要的子类 Exception（异常）和 Error（错误)。Exception 能被程序本身处理(try-catch)， Error是无法处理的(只能尽量避免)。

    Exception 和 Error 二者都是 Java 异常处理的重要子类，各自都包含大量子类。

    Error

    VirtualMachineError（虚拟机错误）：StackOverFlowError(栈溢出）、OutOfMemoryError（内存溢出）、InternalError（网络错误）、UnknownError（未知错误）

    ThredDeth：线程死亡，无需特别关注。

    Exception

    检查异常

    这种异常都发生在编译阶段，Java编译器强制程序去捕获此类异常，比如最常见的IO异常和SQL异常以及ClassNotFoundException。

    此类异常要求使用Try，catch进行处理，一般在以下情况下使用：

    （1）异常的发生并不会导致程序出错，进行处理后可以继续执行后续的操作，例如，当连接数据库失败后，可以重新连接后进行后续操作。

    （2）程序依赖于不可靠的外部条件， 例如系统IO。

    运行时异常

    （1）对于运行时异常，编译器没有强制对其进行捕获并处理。如果不对这种异常进行处理，仅仅做抛出处理时，会由JVM来处理，例如NullPointerException异常，它就是运行时异常。

    在Java语言中，最常见的运行时异常包括NullPointerException（空指针异常）、ClassCastException（类型转换异常）、ArrayIndexOutOfBoundsException（数组越界异常）、ArrayStoreException（数组存储异常）、BufferOverflowException（缓冲区溢出异常）、ArithmeticException (算术异常）等。

    （2）出现运行时异常后， 系统会把异常一直往上层抛出， 直到遇到处理代码为止。若没有处理块，则抛到最上层；如果是多线程就用 Thread.run ()方法抛出，如果是单线程，就用main()方法抛出。抛出之后，如果是线程，那么这个线程也就退出了。如果是主程序抛出的异常，那么整个程序也就退出了。 

     所以，如果不对运行时的异常进行处理，后果是非常严重的，一旦发生，要么是线程中止，要么是主程序终止。
